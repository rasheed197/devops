# FILE TYPES

## TYPES OF FILES
1. Regular files (-): normal files such as text, data, or executable files

2. Directory (d): files that are lists of other files

3. Link (l): a shortcut that points to the location of the actual file

4. Special file (c): mechanism used for input and output. Such as files in /dev

5. Socket (s): a special file that provides inter-process networking protected by the System's access control

6. Pipe (p): a special file that allows processes to communicate with each other without using network socket semantics

### 1. REGULAR FILE
- Files begins with hyphen "-" i.e "-rw-------"
- Regular file means it could be a text file or a binary file. Example
```
[root@vbox ~]# ls -l

total 8

-rw-------. 1 root root 2027 Oct 10  2024 anaconda-ks.cfg
-rw-------. 1 root root 1388 Oct 10  2024 original-ks.cfg

[root@vbox ~]#

```
- "anaconda-ks.cfg" is a regular file, a text file.
- to be sure, you can run this command

```
[root@vbox ~]# file anaconda-ks.cfg
anaconda-ks.cfg: ASCII text
```
- regular file can also be a binary file. Example
```
[root@vbox ~]# file /bin/tar

/bin/tar: ELF 64-bit LSB pie executable, x86-64, version 1 (SYSV), dynamically linked, interpreter /lib64/ld-linux-x86-64.so.2, BuildID[sha1]=ef9ddc8efa91ccd9b562fb57867deb2832df9cc3, for GNU/Linux 3.2.0, stripped
```

### 2. DIRECTORY
- Directory begins with "d" i.e "drwxr-xr-x"
```
[vagrant@vbox ~]$ ls -l

total 8
drwxr-xr-x. 2 vagrant vagrant    6 Nov  6 06:07 backupdir
drwxr-xr-x. 3 vagrant vagrant   86 Nov  6 06:40 dev
-rw-r--r--. 1 vagrant vagrant  121 Nov  6 08:57 firstfile.txt
drwxr-xr-x. 2 vagrant vagrant 4096 Nov  6 07:57 textdir
[vagrant@vbox ~]$
```

### 3 SPECIAL FILES
- Special files begin with "c"
- "c" for character files like keyboard
- These are found in the "/dev" directory
```
lrwxrwxrwx. 1 root root           4 Nov  6 15:07 rtc -> rtc0
crw-------. 1 root root    250,   0 Nov  6 15:07 rtc0
brw-rw----. 1 root disk      8,   0 Nov  6 15:07 sda
brw-rw----. 1 root disk      8,   1 Nov  6 15:07 sda1
crw-rw-rw-. 1 root tty       5,   0 Nov  6 15:07 tty
crw--w----. 1 root tty       4,   0 Nov  6 15:07 tty0
```

- rtc0 is a character file
- rtc is a link file that points to rtc0
- sda: hard-disk, it's a lock file
- tty: shell session, the terminal


## CREATE LINK FILE
```
syntax: ln -s destination src
```
- -s: soft link

Example 
```
# create nested directories
mkdir -p opt/dev/opd/devops/test

# create and edit a file
vim opt/dev/opd/devops/test/commands.txt

# create link file that points to commands.txt
ln -s opt/dev/opd/devops/test/commands.txt cmds

# read the content of commands.txt
cat cmdS
```

## REMOVE A LINK FILE 
This will only remove the link, the original file still exists.
```
unlink cmds

# OR

rm cmds
```
