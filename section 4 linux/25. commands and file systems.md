# COMMANDS AND FILE SYSTEMS

Login to your vm (CentOS) using Vagrant

### BASIC COMMANDS
```
# see what user you are
whoami

# see where you are
pwd

# see what's in current directory
ls

# read a file
cat [file_path e.g /etc/os-release]

# clear the screen
clear

# go back to user's home directory (method1)
cd

# go back to user's home directory (method2)
cd ~
```

- etc/os-release file contains info about the OS name, version etc
- The shell prompt looks something like this
```
[vagrant@localhost ~] $
```
- "vagrant" is the user i.e username
- "localhost" is the hostname, no hostname is set so default is "localhost"
- "~" means that you're in the home directory of vagrant user
- "$" represents a normal user's shell
- only vagrant user has permission to switch to root user

### SWITCH TO ROOT USER
```
# switch to root user
sudo -i
```
- The shell prompt now looks something like this
```
[root@localhost ~] #
```
- "root" is the user i.e username
- "localhost" is the hostname, no hostname is set so default is "localhost"
- "~" means that you're in the home directory of root user
- "#" represents a root user's shell
- the home directory of root user is "/root"
- but the root directory is "/"
```
# root directory
/

# root user home directory
/root

# navigate to root directory
cd /

# see all files in root directory
ls /

# go back to user's home directory 
cd
```

## ROOT DIRECTORY (/)
- "/" is the top level directory of the Linux OS
- It contains all the files in the Linux OS
